# Node Type Perks

## 1. Purpose

Node Type Perks make two nodes of the same Exploration Level feel different
because of decisions specific to their fantasy.

They are separate from Node Specialization:

| System | Scope | Question answered |
|---|---|---|
| Specialization | Shared by every node type | Quantity, Discovery, or Logistics? |
| Type Perk | Unique to Mining/Farming/etc. | How does this particular node operate? |

Specialization changes broad efficiency. Type Perks add rules, targeting,
event choices, collection patterns, and type-specific utility.

## 2. Unlock structure

Each node earns four Perk Points:

| Exploration Level | Perk tier | Theme |
|---:|---|---|
| 15 | Foundation | Passive identity |
| 35 | Operations | Active management/events |
| 60 | Direction | Resource family preference |
| 85 | Legacy | Mastery mechanic |

At each tier, choose one of three perks. The player therefore owns four perks
at Lv.85 and sees the complete build before the Lv.100 Capstone.

Rules:

- Perks belong to the individual node, not the account.
- Type conversion clears incompatible perks and refunds their Perk Points.
- Tier upgrade never clears perks.
- No perk raises a rare-resource hard cap.
- No perk grants Credits, tradable premium goods, or guaranteed Jackpot.
- Perks remain active after Lv.100.

## 8. Perk synergy with Specialization

Perks may complement a Specialization but must not require one.

Examples:

- Mining Reinforced Shaft + Logistics creates a low-maintenance mine.
- Farming Crop Rotation + Discovery creates a research-oriented farm.
- Wood Master Carpenter + Industry creates a server-project supplier.
- Livestock Ranch Hand + Discovery creates a worker-training ranch.
- Hunter Apex Contract + Discovery creates an event-focused guild hall.

Cap combined effects:

- Total passive quantity from perks: no more than +20%.
- Research gain from perks: no more than +15%, excluding catch-up/rested rules.
- Event duration reduction from all perks/synergies: no more than 35%.
- Perks never increase daily/weekly rare caps.

## 9. Respec

- Perk preview is free.
- A newly selected perk has a 24-hour free-change window.
- After that, respec one tier at a time with Coins.
- Cost: `500 + 25 × Node Exploration Level`.
- Seven-day cooldown for the changed tier.
- Hybrid Pay may offset at most 15%.
- Respec never removes Journal, Achievement, or Node EXP progress.

Type conversion:

- Show all lost type-specific selections.
- Refund four unspent Perk Points on the converted node.
- New type perks must be selected again.
- No Coin refund for past respecs.

## 10. UI

Node Detail has separate buttons:

- `Specialization`: Quantity/Discovery/Logistics.
- `Type Perks`: the four node-specific milestones.

Perk screen:

- Vertical timeline at Lv.15/35/60/85.
- Selected choice and two alternatives.
- Exact mechanical effect.
- Synergy note with current Specialization.
- Next Perk Level and estimated date.
- Preview mode before confirmation.

Level-up notification at 15/35/60/85 opens a reward card but does not force an
immediate decision. Unspent points remain visible in the Hub.

## 11. Achievements

Perk achievements recognize decisions without requiring every branch:

- Unlock the first Type Perk.
- Trigger the selected perk mechanic ten times.
- Reach Lv.100 with four selected perks.
- Complete an event using a perk/specialization synergy.

There is no achievement for repeatedly respeccing or owning every mutually
exclusive perk.

## 12. Technical model

Suggested definitions:

`plugins/Idle/node-perks.yml`

```yaml
mining_ore_sense:
  node-type: MINING
  unlock-level: 15
  tier: FOUNDATION
  group: mining_foundation
  effect: ORE_PITY
  params:
    rolls: 30
    eligible: [RAW_COPPER, RAW_IRON, RAW_GOLD]
```

Suggested persistence:

```text
idle_node_perks
  node_id
  perk_id
  selected_at
  free_change_until
  cooldown_until
  PRIMARY KEY (node_id, perk_id)
```

Runtime effects subscribe to production, collection, event, project, and
journal domain events. GUI code does not apply perk effects directly.
